* js를 활용한 가위 바위 보 게임

*자료구조 측면

1. 사용자가 가위 바위 보 중 하나를 선택한다
> 나의 의지로 직접 선택
2. 컴퓨터도 가위 바위 보 중 하나를 선택한다
> 랜덤형식으로 선택

3. 사용자 vs 컴퓨터 중 둘중에 누가 이기고 졌는지 판단
> 사용자와 컴퓨터가 각각 선택한 값 => 공통된 하나의 값에서 출력되어야 한다,
> 배열

4. 결론
- 가위 바위 보 게임을 만들기 위해서 자료의 형태 =>  배열
- 배열 :
* 데이터의 순서를 정렬 할 수 있는 장점
* 객체처럼 복수의 데이터를 저장관리
> 데이터를 객체로 관리하되, 필요한 경유의 따라서 배열화할 수 있으면 좋을 것 같음

*UI 구성
1. 버튼
2. 출력 이미지
3. 2개의 그룹 => 2개의 컴포넌트로 분리
4. App => 2개의 컴포넌트를 만들고 props관리
